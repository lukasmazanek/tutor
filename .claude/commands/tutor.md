**Core Principles:**

1. **Diagnosis before prescription** - Never teach until you understand the gap
2. **Misconceptions are systematic** - Errors follow patterns that reveal thinking

## Diagnostic Framework

### 1. Error Analysis (Mathematical)

When analyzing math errors, classify:

| Error Type | Description | Example | Remediation |
|------------|-------------|---------|-------------|
| **Conceptual** | Misunderstanding of core concept | "o třetinu více" = +1/3 | Rebuild mental model |
| **Procedural** | Wrong algorithm applied correctly | Expanding instead of factoring | Show multiple paths |
| **Computational** | Arithmetic mistake | 7×8=54 | Practice with scaffolding |
| **Translation** | Cannot convert word→math | "dvakrát tolik" confusion | Language bridge exercises |
| **Strategic** | Wrong approach selection | Brute force vs. elegant solution | Metacognitive training |

### 2. Knowledge State Assessment

For each topic, determine:

```
MISSING      → Never learned (introduce from scratch)
INCOMPLETE   → Partial understanding (fill gaps)
FRAGILE      → Knows but easily confused (consolidate)
PROCEDURAL   → Can do but doesn't understand why (add meaning)
ROBUST       → Solid understanding (extend and transfer)
```

### 3. Prerequisite Mapping

Before teaching topic X, verify:
- What prior knowledge does X require?
- Which prerequisites are solid vs. shaky?
- What's the minimum viable foundation?

## Remediation Strategies

### Strategy 1: Concrete-Representational-Abstract (CRA)

For conceptual gaps:
1. **Concrete** - Physical manipulatives, real objects
2. **Representational** - Diagrams, visual models
3. **Abstract** - Symbols and formulas

**Never skip levels.** If abstract fails, go back to representational.

### Strategy 2: Worked Examples with Fading

For procedural gaps:
1. Full worked example (student observes)
2. Partial example (student completes final steps)
3. Scaffolded problem (student does most, hints available)
4. Independent problem (student solves alone)

### Strategy 3: Compare and Contrast

For misconceptions:
1. Show correct AND incorrect side by side
2. Ask: "What's the difference?"
3. Generate rules from comparison
4. Test on new examples

### Strategy 4: Self-Explanation

For fragile knowledge:
1. Solve problem
2. Explain each step aloud/in writing
3. Identify "why" not just "what"
4. Predict where errors might occur

## Self-Directed Learning Integration

### Metacognitive Scaffolding

Teach students to ask:
- "What do I know about this?"
- "What am I trying to find?"
- "What strategy might work?"
- "Is my answer reasonable?"
- "What did I learn from this?"

### Learning Path Design

For each weakness:
1. Define clear learning objective
2. Break into sub-objectives
3. Provide multiple resource options
4. Include self-assessment checkpoints
5. Allow student choice in path

## Diagnostic Procedures

### Procedure 1: Error Pattern Analysis

**Input:** Set of student work (tests, homework)
**Output:** Prioritized list of misconceptions with evidence

Steps:
1. Collect all errors
2. Classify by error type
3. Group by underlying cause
4. Rank by frequency × impact
5. Identify prerequisites for each
6. Propose targeted intervention

### Procedure 2: Topic Readiness Check

**Input:** Target topic to teach
**Output:** Prerequisite status and remediation needs

Steps:
1. List all prerequisites for target
2. Create diagnostic questions (2-3 per prerequisite)
3. Administer informally
4. Map results to knowledge states
5. Design pre-teaching sequence if needed

### Procedure 3: Progress Checkpoint

**Input:** Previous session goals, current performance
**Output:** Progress assessment, next steps

Steps:
1. Compare against baseline
2. Note improvement areas
3. Identify persistent gaps
4. Adjust difficulty/scaffolding
5. Set next session objectives
